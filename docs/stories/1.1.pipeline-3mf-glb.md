## Story
**As a** lojista,
**I want** um processo documentado que converta meu 3MF do Bambu Studio em GLB otimizado para web com âncoras de personalização,
**so that** cada novo produto entre na loja leve, rápido e já preparado para a geração automática de produção (Épico 5).

## Acceptance Criteria
1. Processo reproduzível e documentado (`docs/guides/onboarding-de-produto.md`): 3MF → malhas separadas por parte colorível → GLB com compressão Draco ou Meshopt
2. GLB do comedouro 15cm com ≤ 4 MB (meta 2 MB), validado no gltf-validator sem erros
3. Âncora nomeada `name_slot` no GLB (empty/node) com posição, rotação e escala onde o texto do nome será renderizado
4. Fonte do texto identificada (a mesma usada no Bambu Studio) e exportada como asset web (woff2 + arquivo de fonte para o Épico 5); registrada no `param_schema`
5. Metadados de personalização (âncora, fonte, curvatura, limites de caracteres) armazenados em formato JSON versionado junto ao asset — contrato que o Épico 5 consumirá (PRD risco #7)
6. Asset original movido de `ARQUIVO DIMENCIONADO com nome centralizado 15cm.3mf` (raiz) para `assets/models/comedouro-pet/15cm.3mf`; GLB publicado no bucket do storage

## Dev Notes
- O 3MF é o arquivo de PRODUÇÃO; o GLB é só para web (PRD §5 nota técnica). Nunca degradar/substituir o 3MF original.
- O 3MF real está na raiz do repo: `ARQUIVO DIMENCIONADO com nome centralizado 15cm.3mf`.
- Malha separada por parte colorível é o que permite V-03 (troca de cor por mesh target).
- A âncora + fonte definidas aqui são o contrato de longo prazo com o Épico 5 (mesma fonte, mesma posição = risco #7 mitigado). Não improvisar nomes: usar `name_slot` como no PRD §8.
